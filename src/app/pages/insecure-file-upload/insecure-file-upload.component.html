📁 File Upload Risks

📚 Что это за уязвимость?

Уязвимость возникает, когда приложение позволяет загружать файлы без строгой проверки типа, содержимого и места хранения.

👉 Это может привести к загрузке и выполнению вредоносного кода.

📤 Загрузка файла

Попробуй загрузить файл (в демо нет реального backend).

Type: {{ file.type }}

💥 Как происходит атака

  1. Злоумышленник загружает файл (например: .html или .svg)
  2. Файл попадает в публичную директорию
  3. Браузер начинает его исполнять
  4. Выполняется JS код внутри файла

Simulated payload example:

<svg onload="alert('XSS via file upload')" />

🧪 Preview (simulated unsafe rendering)

💡 Реальные кейсы

  • 🧨 Upload .svg → XSS через onload/onerror
  • 💻 Upload .html → execution in browser
  • 📂 Upload .js → execution if served as static
  • 🗂 Upload to public bucket (S3 misconfig)

🛡️ Как защититься

  • ✔ whitelist MIME types (image/png, etc.)
  • ✔ server-side validation (НЕ frontend only)
  • ✔ rename files on upload
  • ✔ store outside web root
  • ✔ disable execution in static storage
