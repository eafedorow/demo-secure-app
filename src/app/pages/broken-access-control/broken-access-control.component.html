🔐 Broken Access Control

📚 Что это за уязвимость?

Broken Access Control возникает, когда приложение показывает или скрывает данные только на уровне UI, но не проверяет доступ на уровне логики или backend.

👉 Это значит: пользователь может получить доступ к данным, даже если интерфейс это “запрещает”.

🎭 Контекст пользователя

Сейчас мы симулируем роли пользователей в системе.

Текущая роль: {{ role }}

⚠️ Как происходит атака

  1. Пользователь заходит в систему как обычный USER
  2. UI скрывает админ-кнопки через *ngIf
  3. Но данные уже загружены в frontend
  4. Пользователь меняет роль в DevTools или URL
  5. И получает доступ к запрещённым данным

💡 Реальные кейсы из продакшена

  • 👀 Пользователь меняет ID в URL → получает чужие данные (IDOR)
  • 🧑‍💻 Открывает /admin напрямую без проверки
  • 🔓 API возвращает данные без проверки роли
  • 🧾 Frontend скрывает кнопку, но endpoint доступен

🛡️ Как защищаться

  • ✔ Проверять доступ на backend (обязательно)
  • ✔ Использовать RBAC/ABAC на сервере
  • ✔ Никогда не доверять UI
  • ✔ Валидировать каждый запрос
  • ✔ Защищать API, а не кнопки
