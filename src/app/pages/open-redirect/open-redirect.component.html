🔓 Open Redirect

📚 Что это за уязвимость?

Open Redirect возникает, когда приложение перенаправляет пользователя на URL, который передаётся извне (query params, state, backend).

👉 Если URL не валидируется — злоумышленник может отправить пользователя на любой сайт.

🎯 Как это работает

  1. Пользователь нажимает “Login”
  2. Приложение читает параметр redirect
  3. И делает window.location.href = redirect
  4. Без проверки домена

💥 Симуляция Open Redirect

Здесь показано 2 подхода: опасный (unsafe) и безопасный (safe).

❌ Unsafe redirect

Код:

window.location.href = redirectUrl;

⚠️ Приложение полностью доверяет пользовательскому вводу.

Это означает, что злоумышленник может передать:

https://evil-site.com

👉 Пользователь будет перенаправлен на внешний ресурс без предупреждения.

🛡 Safe redirect

Код:

              `const allowed = ['/dashboard', '/main'];

              const isValid = allowed.some(path =>
                redirectUrl.startsWith(path)
              );

              if (isValid) window.location.href = redirectUrl;

🛡 Здесь добавлена валидация redirect URL.

Разрешены только внутренние маршруты приложения.

👉 Любая попытка перейти на внешний домен блокируется.

🟢 URL проверяется → разрешены только внутренние маршруты

💡 Реальные кейсы атак

  • 🎣 Фишинг через доверенный домен
  • 🔐 Перенаправление после логина на вредоносный сайт
  • 📧 Email ссылки с подменой redirect
  • 💳 Подделка платежных страниц

🛡️ Как защититься

  • ✔ Разрешать только internal routes
  • ✔ Использовать whitelist доменов
  • ✔ Проверять URL через parser
  • ✔ Не использовать window.location напрямую
