🔐 JWT Storage & Authentication Risks

📚 Что это за проблема?

JWT (JSON Web Token) часто используется для авторизации пользователей.

Проблема возникает не в самом JWT, а в том, где и как он хранится на фронтенде.

👉 Если токен хранится в localStorage или sessionStorage, он становится доступен для JavaScript-кода.

💥 Как происходит атака

  1. Токен хранится в localStorage
  2. В приложение попадает XSS
  3. Злоумышленник выполняет: localStorage.getItem('token')
  4. Токен отправляется на внешний сервер

💡 Реальные кейсы

  • 👀 XSS → кража токена из localStorage
  • 🔁 Refresh token перехватывается при отсутствии httpOnly cookie
  • 🧾 SPA сохраняет access token навсегда
  • 🌐 Токен доступен через DevTools в любой момент

🛡️ Как правильно делать

  • ✔ Использовать httpOnly cookies для refresh token
  • ✔ Access token хранить в memory (не persistent storage)
  • ✔ Делать короткий TTL токенов
  • ✔ Использовать refresh rotation
  • ✔ Минимизировать XSS поверхность
